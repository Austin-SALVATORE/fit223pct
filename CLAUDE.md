# Claude Development Rules

## Mission

Build a premium long-term personal training operating system.

## Priorities

1.  User experience
2.  Reliability
3.  Maintainability
4.  Performance
5.  Visual polish

## Working Process

Before coding: - Understand the requirement. - Check existing
architecture. - Prefer improving existing components over creating
duplicates.

After coding: - Refactor. - Run lint/tests/build where applicable. -
Review UX. - Remove dead code.

## Product Rules

-   Do not introduce unnecessary complexity.
-   Avoid placeholder implementations.
-   Mobile-first.
-   Offline-friendly.
-   Local-first data.
-   Accessibility matters.

## Design Rules

Every screen should answer: - What is happening? - What should I do? -
What happens next?

## Fitness Rules

Follow evidence-informed principles. Never fabricate scientific claims.
Never promise specific body transformation outcomes.

## Documentation

Keep docs concise and current. Update docs whenever architecture or
behavior changes.

## Definition of Done

A feature is complete only when: - Works correctly - Feels polished - Is
responsive - Is accessible - Fits the product vision
